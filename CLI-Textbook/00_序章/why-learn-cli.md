# なぜCLIを学ぶのか - AI時代の重要性

## AI時代だからこそ、CLIが重要な理由

### 1. AIの提案を安全に理解・検証できる

現代のAIツール（ChatGPT、GitHub Copilot、Cursor、Codexなど）は、コマンドやスクリプトを提案してくれます。

**しかし、AIの提案を盲目的に信じるのは危険です。**

#### 例: AIが提案したコマンド
```bash
rm -rf /
```

これは「**バルスコマンド**」と呼ばれ、**すべてのファイルを削除**してしまいます！

**CLIを理解していれば**:
- `rm`: remove（削除）
- `-r`: recursive（再帰的 = フォルダごと）
- `-f`: force（強制 = 確認なし）
- `/`: ルートディレクトリ（すべてのファイル）

→ 「これは危険だ！」と気づけます。

### 2. AIに正確な指示を出せる

AIに作業を依頼するとき、CLIの知識があれば、より正確で具体的な指示ができます。

#### ❌ 曖昧な指示
「最近変更されたファイルを見つけて」

#### ✅ 明確な指示
「過去7日間に変更された.txtファイルを、更新日時順に表示して」

```powershell
# Windows PowerShell
Get-ChildItem -Recurse -Filter "*.txt" |
  Where-Object {$_.LastWriteTime -gt (Get-Date).AddDays(-7)} |
  Sort-Object LastWriteTime -Descending
```

### 3. パイプライン（|）で複雑な処理を自動化

CLIの真の力は、**パイプライン**にあります。

#### パイプライン（|）とは
複数のコマンドを連結して、データを次々と処理する仕組みです。

#### 例: ログファイルからエラーを抽出して集計
```bash
# macOS/Linux
cat application.log | grep "ERROR" | sort | uniq -c | sort -rn
```

**何をしているか**:
1. `cat application.log`: ログファイルを読む
2. `| grep "ERROR"`: 「ERROR」を含む行だけ抽出
3. `| sort`: アルファベット順に並べる
4. `| uniq -c`: 重複を削除して回数を数える
5. `| sort -rn`: 回数の多い順に並べる

**結果**:
```
  45 ERROR: Database connection failed
  23 ERROR: Invalid user credentials
  12 ERROR: File not found
```

**GUIでこれをやろうとすると**:
1. ログファイルを開く
2. 手作業で「ERROR」を検索
3. Excelにコピー
4. 重複を削除
5. 集計
6. ソート

→ CLIなら一行で完了！

## AI時代のCLI学習の意義

### AIとCLIの関係性

```
┌─────────────┐
│   あなた    │ ← CLIスキルで正確な指示
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  AIツール   │ ← コマンド・スクリプトを提案
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ CLI/シェル  │ ← 理解していれば安全に実行
└─────────────┘
```

### 実務での活用例

#### ケーススタディ: LLMコーディングエージェント活用

ある開発プロジェクトでの経験（実例）:

**プロジェクト**: HubSpot CRMからTwenty CRMへの移行
**期間**: 予定1週間 → 実際3週間
**使用ツール**: Codex（GPT-5）、Gemini、Claude

**学んだこと**:
1. **AIに任せられるのは「理解している内容の実装」のみ**
2. **実装方針の決定は人間の責任**
3. **テストがしっかりしていればAIは最強のツール**

**CLI知識が必要だった場面**:
- Git操作（ブランチ、コミット、マージ）
- テスト実行とデバッグ
- ファイル検索と一括変更
- ログ分析

## 安全に学ぶために

### ⚠️ 学習時の注意点

1. **危険なコマンドを知る**: `rm -rf`、`dd`など
2. **バックアップを取る**: 重要なファイルは保護
3. **テスト環境で練習**: 本番環境でいきなり実行しない
4. **確認オプションを使う**: `-Confirm`、`-i`など
5. **ドキュメントを読む**: `--help`、`man`コマンド

### 💡 学習のコツ

1. **小さく始める**: 簡単なコマンドから
2. **毎日少しずつ**: 継続が力になる
3. **実際の問題に適用**: 自分の作業を効率化
4. **失敗を恐れない**: エラーから学ぶ
5. **記録を残す**: よく使うコマンドをメモ
