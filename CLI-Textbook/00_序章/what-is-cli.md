# CLIとは何か

## コマンドラインインターフェース（CLI）の基本概念

### GUI vs CLI

コンピュータとのやり取りには、大きく分けて2つの方法があります。

#### GUI（Graphical User Interface）
- **操作方法**: マウスやタッチ操作
- **例**: エクスプローラー（Windows）、Finder（macOS）
- **特徴**: 視覚的で直感的、初心者に優しい

#### CLI（Command Line Interface）
- **操作方法**: 文字（コマンド）を入力
- **例**: PowerShell（Windows）、Terminal（macOS/Linux）
- **特徴**: 効率的、自動化可能、正確性が高い

### レストランで例えると

**GUIの場合**:
```
あなた: （メニューを指さして）「これください」
店員: 「かしこまりました」
```

**CLIの場合**:
```
あなた: 「カルボナーラ 大盛り チーズ多め」
店員: 「カルボナーラ、大盛り、チーズ多めですね。承りました」
```

CLIでは、正確に「何をしたいか」を言葉で伝えます。

## CLIの構成要素

### 1. シェル（Shell）
コマンドを受け取って実行するプログラムです。

**主なシェル**:
- **Windows**: PowerShell、コマンドプロンプト
- **macOS**: zsh（デフォルト）、bash
- **Linux**: bash、zsh、fishなど

### 2. ターミナル（Terminal）
シェルを操作するためのウィンドウです。

### 3. コマンド（Command）
コンピュータに指示を出すための命令です。

## コマンドの基本構造

```
コマンド名 オプション 引数
```

### 例1: ファイル一覧表示（Windows）
```powershell
Get-ChildItem -Path C:\Users
```

- **コマンド名**: `Get-ChildItem`（ファイル一覧を取得）
- **オプション**: `-Path`（どこの？）
- **引数**: `C:\Users`（Usersフォルダ）

### 例2: ファイル一覧表示（macOS/Linux）
```bash
ls -l /Users
```

- **コマンド名**: `ls`（list - 一覧表示）
- **オプション**: `-l`（詳細表示）
- **引数**: `/Users`（Usersディレクトリ）

## CLIの特徴

### ✅ メリット

1. **効率的**: 複雑な操作を一行で実行可能
2. **自動化可能**: スクリプトで繰り返し作業を自動化
3. **正確**: 実行内容が明確で記録に残る
4. **リモート操作**: ネットワーク越しでもコマンドで操作可能
5. **軽量**: GUIより少ないリソースで動作

### ⚠️ デメリット

1. **学習曲線**: コマンドを覚える必要がある
2. **エラーのリスク**: 誤ったコマンドで重大な影響も
3. **視覚性が低い**: ファイルの内容が直接見えない

## CUI vs CLI

**CUI（Character User Interface）**: 文字ベースのインターフェース全般
**CLI（Command Line Interface）**: CUIの一種で、コマンドで操作するもの

```
CUI（文字ベース全般）
 └── CLI（コマンド操作）
```

## 実際の使用例

### ファイル検索（GUI）
1. エクスプローラーを開く
2. フォルダを一つずつ開いて探す
3. 検索ボックスにファイル名を入力
4. 結果から目的のファイルを探す

### ファイル検索（CLI）
```powershell
# Windows
Get-ChildItem -Recurse -Filter "*.txt"

# macOS/Linux
find . -name "*.txt"
```

一行で、すべての.txtファイルを検索完了！

## まとめ

- CLIは**文字でコンピュータに指示を出す方法**
- GUIより**効率的で正確**だが、**学習が必要**
- **シェル**がコマンドを実行し、**ターミナル**で操作する
- コマンドは `コマンド名 オプション 引数` の形式
